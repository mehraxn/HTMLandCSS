What position: relative really does

The best way to understand relative: the element keeps its original space in the layout, then it is visually shifted from that spot using top, left, right, or bottom.

Think of it like this

What people often imagine

“The box moves, so the layout must recalculate and everything slides with it.”

What actually happens

“The layout keeps the box's original slot. The box is then nudged away from that slot.”

Why this matters in real layouts

Header
Relative box
Next content

The “Next content” block does not move up to replace the relative box's original place. That is the key idea: relative changes where the element is drawn, not where its space exists in layout.

The exact CSS idea

.box {
  position: relative;
  top: 24px;
  left: 34px;
}
  • The box still owns its original place.
  • top and left move it from that place.
  • Nearby elements behave as if the box never left its original slot.
  • This is why relative is great for small nudges and for creating a positioning context.
