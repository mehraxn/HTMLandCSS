Where position: relative shows up in shopping websites

In real e-commerce pages, relative is often added to a card, image, or button so small UI pieces like badges, wishlist hearts, labels, and quick actions can be placed exactly where they belong.

Common shop patterns

Places where stores use relative positioning

1. Sale badge on a card

The badge is absolutely positioned, and the product card is relative so the badge stays pinned to that card.

.product-card {
  position: relative;
}

.product-badge {
  position: absolute;
  top: 12px;
  left: 12px;
}

2. Wishlist heart over the image

The heart is placed on the top-right corner of the card without changing the card’s normal place in the grid.

.wish-btn {
  position: absolute;
  top: 12px;
  right: 12px;
}

3. Quick view button near the bottom

The button is anchored inside the same card, so it moves together with the card on responsive layouts.

.quick-view {
  position: absolute;
  bottom: 12px;
  right: 12px;
}

Best mental model

In a real shop page, position: relative often means: “Keep this box in the normal layout, but let absolutely positioned children use this box as their anchor.”
