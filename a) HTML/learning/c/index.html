CSS Positioning Schemes

A complete visual guide to static, relative, absolute, and fixed positioning.

Static Relative Absolute Fixed Compare

Quick summary

  • Static: default position, follows the normal document flow.
  • Relative: stays in the normal flow, then moves relative to its original position.
  • Absolute: removed from the normal flow and positioned relative to the nearest positioned ancestor.
  • Fixed: removed from the normal flow and positioned relative to the viewport.

1) Static positioning

position: static is the default for most elements. The element stays in the normal document flow, and properties like top, right, bottom, and left do not affect it. 

.box {
  position: static;
}

Use this when you want standard layout behavior.

2) Relative positioning

A relatively positioned element keeps its original space in the layout, but it can be shifted visually using top, left, right, or bottom. 

.moved {
  position: relative;
  top: 18px;
  left: 24px;
}

Important: the original space is still reserved, even after the element moves.

3) Absolute positioning

An absolutely positioned element is removed from the normal flow. It is positioned using offset properties, relative to the nearest ancestor that is not static. 

.parent {
  position: relative;
}

.child {
  position: absolute;
  top: 12px;
  right: 12px;
}

If no positioned ancestor exists, the element is positioned relative to the page.

4) Fixed positioning

A fixed element is also removed from the normal flow, but unlike absolute positioning, it is positioned relative to the viewport instead of a parent container. 

.help-widget {
  position: fixed;
  bottom: 24px;
  right: 24px;
}

Common uses: sticky chat buttons, floating action buttons, cookie banners, and pinned toolbars.

Best practices

Use relative sparingly

Relative is great for tiny adjustments and for creating a containing block for absolute children.

Use absolute intentionally

Absolute positioning is powerful, but it can overlap content if the parent and spacing are not planned well.

Use fixed for persistent controls

Keep fixed elements small and unobtrusive so they do not cover important content on smaller screens.
